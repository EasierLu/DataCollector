# DataCollector 数据收集器 - 需求文档

## 1. 项目概述

### 1.1 项目背景
基于 HTTP/HTTPS 协议的通用数据收集系统，用于收集表单/用户提交的数据。

### 1.2 技术栈
- **后端语言**: Go 1.21+
- **Web框架**: Gin
- **数据库**: SQLite (默认) / PostgreSQL / MySQL
- **前端**: HTML + TailwindCSS + Alpine.js
- **实时通信**: WebSocket / Server-Sent Events
- **跨平台**: Windows / macOS / Linux 原生支持
- **部署方式**: 原生二进制 / Docker 容器

## 2. 功能需求

### 2.1 数据采集模块 (Collector)

| 需求编号 | 需求描述 | 优先级 | 备注 |
|---------|---------|-------|------|
| COL-001 | 提供 RESTful API 接收数据提交 | 高 | POST /api/v1/collect/:source_id |
| COL-002 | 支持 JSON 格式数据提交 | 高 | Content-Type: application/json |
| COL-003 | 支持 Form-data 格式数据提交 | 中 | Content-Type: multipart/form-data |
| COL-004 | 数据字段验证（必填、类型、长度） | 高 | 基于数据源配置动态验证 |
| COL-005 | 请求频率限制（Rate Limiting） | 高 | 按 Token 维度限流，默认每分钟 100 次；同时支持按 IP 维度限流（默认每分钟 200 次），可通过配置调整 |
| COL-006 | 请求体大小限制 | 中 | 最大 1MB |
| COL-007 | CORS 跨域支持 | 中 | 可配置允许的域名 |
| COL-008 | 记录客户端 IP 和 User-Agent | 中 | 用于审计和防刷 |
| COL-009 | **Token 认证** | 高 | Header: X-Data-Token |
| COL-010 | 批量数据提交 | 低 | 一次提交多条记录 |

## 3. 数据库模型

### 3.1 用户表 (users)
```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username VARCHAR(50) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    role VARCHAR(20) DEFAULT 'user', -- admin, user
    status INTEGER DEFAULT 1, -- 0:禁用, 1:启用
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 3.2 数据源表 (data_sources)
```sql
CREATE TABLE data_sources (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    schema_config TEXT, -- JSON 格式存储字段定义和验证规则；SQLite 以 TEXT 存储，PostgreSQL/MySQL 使用原生 JSON 类型
    status INTEGER DEFAULT 1, -- 0:禁用, 1:启用
    created_by INTEGER,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (created_by) REFERENCES users(id)
);
```

### 3.3 数据令牌表 (data_tokens) - 新增
```sql
CREATE TABLE data_tokens (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    source_id INTEGER NOT NULL,
    token_hash VARCHAR(64) UNIQUE NOT NULL,  -- 存储 Token 的哈希值
    name VARCHAR(100),  -- Token 名称/备注
    status INTEGER DEFAULT 1, -- 0:禁用, 1:启用
    expires_at DATETIME,  -- 过期时间，NULL 表示永不过期
    last_used_at DATETIME,  -- 最后使用时间
    created_by INTEGER,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (source_id) REFERENCES data_sources(id),
    FOREIGN KEY (created_by) REFERENCES users(id)
);
```

### 3.4 数据记录表 (data_records)
```sql
CREATE TABLE data_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    source_id INTEGER NOT NULL,
    token_id INTEGER,  -- 记录使用的哪个 Token 提交
    data TEXT NOT NULL, -- JSON 格式存储提交的数据；SQLite 以 TEXT 存储，PostgreSQL/MySQL 使用原生 JSON 类型
    ip_address VARCHAR(45),
    user_agent TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (source_id) REFERENCES data_sources(id),
    FOREIGN KEY (token_id) REFERENCES data_tokens(id)
);
```

### 3.5 统计表 (statistics)
```sql
CREATE TABLE statistics (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    source_id INTEGER,
    stat_date DATE NOT NULL,
    count INTEGER DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (source_id) REFERENCES data_sources(id)
);
```

### 3.6 系统配置表 (system_configs) - 新增
```sql
CREATE TABLE system_configs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    config_key VARCHAR(100) UNIQUE NOT NULL,
    config_value TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 存储初始化状态、数据库配置、系统设置等
-- 如: initialized=true, db_driver=sqlite, server_port=8080
```

## 4. API 接口设计

### 4.1 数据采集接口

#### 提交数据
```
POST /api/v1/collect/:source_id
Headers:
  X-Data-Token: <token>  # 用户在后台生成的数据提交 Token
  Content-Type: application/json

Body:
{
  "field1": "value1",
  "field2": "value2"
}

Response 200:
{
  "code": 0,
  "message": "success",
  "data": {
    "record_id": 12345
  }
}

Response 401:
{
  "code": 1000,
  "message": "invalid or expired token"
}

Response 403:
{
  "code": 1001,
  "message": "token disabled"
}

Response 400:
{
  "code": 1002,
  "message": "validation failed",
  "errors": {
    "field1": "required"
  }
}

Response 429:
{
  "code": 1003,
  "message": "rate limit exceeded"
}
```

## 6. 非功能需求

### 6.1 性能要求
- API 响应时间 < 200ms (P95)
- 支持并发 1000+ 请求
- 数据库查询优化，单表支持百万级数据

### 6.2 安全要求
- HTTPS 支持
- SQL 注入防护
- XSS 防护
- CSRF 防护
- 敏感数据加密

### 6.3 部署要求
- **零配置启动**：首次运行自动进入 Web 初始化向导
- **跨平台支持**：Windows、macOS、Linux 原生二进制
- **单二进制文件部署**：无需依赖，独立运行
- **Docker 容器化支持**：提供官方镜像和 docker-compose 配置
- 配置文件外置（可选）
- 日志文件轮转

### 6.4 可靠性要求
- **优雅关闭**：收到 SIGTERM/SIGINT 信号后，等待正在处理的请求完成（超时 30 秒），再关闭数据库连接和释放资源
- **启动自检**：启动时自动检测数据库连通性，失败时输出明确错误日志并退出
- **Panic 恢复**：全局 Gin Recovery 中间件捕获 panic，防止单个请求崩溃导致进程退出

### 6.5 统一错误码体系

| 错误码范围 | 模块 | 说明 |
|-----------|------|------|
| 0 | 通用 | 成功 |
| 1000-1099 | 数据采集 | Token 认证、数据验证、限流相关 |
| 2000-2099 | 用户认证 | 登录失败、Token 过期、权限不足 |
| 3000-3099 | 数据源管理 | 创建/编辑/删除数据源相关 |
| 4000-4099 | 数据查询 | 查询参数错误、导出失败 |
| 5000-5099 | 系统运维 | 健康检查、初始化相关 |
| 9000-9099 | 通用错误 | 参数缺失、内部错误、未知错误 |

所有 API 响应统一使用以下格式：
```json
{
  "code": 0,
  "message": "描述信息",
  "data": {},
  "errors": {}
}
```

### 6.6 数据保留与清理策略
- 数据记录默认**永久保留**，管理员可在系统设置中配置自动清理策略
- 可配置项：保留天数（如 90 天、180 天、365 天）、按数据源单独设置
- 清理方式：后台定时任务，软删除后延迟物理删除
- 统计表数据独立于原始数据保留，清理数据记录不影响历史统计

### 6.7 日志规范
- **日志格式**：结构化 JSON 日志，包含 timestamp、level、module、message、trace_id 字段
- **日志分级**：DEBUG / INFO / WARN / ERROR，生产环境默认 INFO
- **审计日志**：管理后台的关键操作（登录、创建/删除数据源、生成/禁用 Token、数据导出）记录独立审计日志
- **请求日志**：记录每个 HTTP 请求的方法、路径、状态码、耗时、客户端 IP
- **日志轮转**：按大小（默认 100MB）和时间（默认每日）自动轮转，保留最近 30 天

### 6.8 API 版本策略
- 当前版本 v1，通过 URL 路径前缀 `/api/v1/` 区分
- 破坏性变更（字段删除、类型变更、语义变化）必须升级版本号（如 v2）
- 非破坏性变更（新增字段、新增端点）可在当前版本内迭代
- 旧版本 API 在新版发布后至少保留 6 个月

## 附录 A: Token 机制说明

### 生成流程
1. 用户在管理后台点击"生成 Token"
2. 系统生成随机字符串（格式：`dt_` + 32位随机字符）
3. 系统计算 Token 的哈希值（SHA-256）存储到数据库
4. **仅将明文 Token 展示给用户一次**
5. 后续验证时，将用户提交的 Token 哈希后与数据库存储的哈希值比对

### 安全特性
- **一次性可见**：明文 Token 仅在生成时展示，之后无法找回
- **哈希存储**：数据库只存储 Token 的哈希值，即使数据库泄露也无法直接使用
- **独立控制**：每个 Token 可单独禁用/删除，不影响其他 Token
- **使用追踪**：记录 Token 的最后使用时间，便于审计

### 使用示例
```bash
# 提交数据时携带 Token
curl -X POST http://localhost:8080/api/v1/collect/1 \
  -H "X-Data-Token: dt_a1b2c3d4e5f6..." \
  -H "Content-Type: application/json" \
  -d '{"email": "user@example.com", "message": "Hello"}'
```
