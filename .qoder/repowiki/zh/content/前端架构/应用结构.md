## 简介
本文件面向DataCollector前端应用，系统性梳理Vue 3 + TypeScript + Vite构建的单页应用（SPA）结构与启动流程，重点覆盖以下方面：
- 应用初始化：应用创建、插件安装、全局配置
- Element Plus UI库集成与国际化配置
- 样式体系组织与CSS变量使用
- 应用入口点配置与启动流程
- 第三方库集成与版本管理策略
- 性能优化与最佳实践

## 项目结构

### 认证与Token刷新机制
- 登录与登出
  - auth.ts封装登录请求、Token写入localStorage、登出清理与路由跳转
- Token刷新
  - useAuth在组件挂载时启动定时器，计算Token剩余有效期并在临界值时调用刷新接口
- 安全与健壮性
  - 刷新失败静默处理，避免阻塞主流程；登出时清除Token并跳转登录页

```mermaid
sequenceDiagram
participant View as "DashboardView"
participant Composable as "useAuth"
participant API as "auth.ts"
participant Store as "auth.ts(Store)"
View->>Composable : 挂载时启动定时器
loop 每5分钟检查一次
Composable->>Composable : 解析Token过期时间
alt 剩余时间小于阈值
Composable->>API : refreshToken()
API-->>Composable : 返回新Token
Composable->>Store : 更新localStorage
else 无需刷新
Composable-->>Composable : 忽略
end
end
View->>Store : logout() 清理Token并跳转登录
```

  - 生产构建输出至dist，结合CDN与缓存策略提升加载速度

## 故障排查指南
